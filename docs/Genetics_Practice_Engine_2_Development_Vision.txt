First Milestone

Before adding more genetics content, verify that the modular
architecture is working correctly.

Checklist:

-   App loads correctly.
-   Dashboard appears.
-   Random Practice works.
-   Topic Practice works.
-   Linkage still functions.
-   No browser console errors.

Once this passes, the architecture is considered stable.

Engine Design

Rather than exposing individual generators, each topic should export a
structured object.

Conceptually:

Topic ↓ Learning Objectives ↓ Reasoning Families ↓ Question Generators

The scheduler should not contain genetics knowledge.

Instead, it simply requests:

Topic ↓ Learning Objective ↓ Reasoning Family ↓ Generated Question

This keeps the architecture clean, modular, and easy to maintain.

------------------------------------------------------------------------

Future Expansion

Once Engine 2.0 is stable, new topics can be added independently, for
example:

-   Population Genetics
-   Hardy–Weinberg
-   Evolution
-   Molecular Biology
-   PCR
-   DNA Sequencing
-   CRISPR

No existing topic should need modification to support these additions.

Long-Term Vision: Instructor Toolkit

Build a feature that allows instructors to assemble custom practice sets
by selecting learning objectives.

Example:

Monohybrid ✓ Reverse inference ✓ Conditional probability ✓ Testcross

Linkage ✓ Coupling ✓ Recombination

Then choose “Generate Assignment”.

The engine creates unique question sets for students.

------------------------------------------------------------------------

Ultimate Goal

Do not build just another question generator.

Build a platform that students consider the best place to practice
undergraduate genetics, and that instructors recognize as a powerful,
flexible teaching resource.
